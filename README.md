Краткое руководство по работе с backend стороной проекта KnowledgeApp(C#).

## I. Как локально запустить проект
#### Необходимые установки
1) Необходим иметь установленный Docker Desktop. Если его нет, установите его любым удобным для вас способом или в зависимости от спецификации вашей ОС. Например, для Windows - https://docs.docker.com/desktop/setup/install/windows-install/. Для Linux воспользуйтесь менеджером пакетов вашей ОС.
2) Установите Git аналогично, если его нет.
3) Установите VS Code аналогично, если его нет.
4) Т.к проект использует технологию ASP.NET, необходимо иметь установленный .NET 8(точнее, 8.0.100). Установка аналогична, однако у некоторых менеджеров пакетов Linux может не быть соответствующей версии .NET, поэтому необходимо искать другие пути её установки.
5) Загрузите через Git данный репозиторий на ваше устройство. По умолчанию в вашем репозитории будет выбрана ветка main. Обычно, именно от неё нужно делать свою ветку, однако перед этим стоит уточнить у других участников проекта, какая ветка является самой актуальной.
#### Процесс запуска
 1) Откройте файл docker-compose.yml в корне репозитория. Нажимаем на него. В VS Code в верхней части файла будет надпись "Run All Services". Нажимаем на неё. В этот момент должен работать Docker. Выполнение команды создаст контейнер с DB.
2) Заходим в репозитории в терминале в директорию **/KnowledgeApp/KnowledgeApp/KnowledgeApp**. В ней последовательно прописываем в терминале **dotnet build**, **dotnet run**. Это запускает сам backend локально, но без DB.
3) Переходим по локальному адресу http://localhost:5134/swagger. Это открывает страницу сервиса **Swagger**, где можно отдельно протестировать endpoint каждой сущности.

Готово, теперь вы можете локально тестировать backend и вносить в него свои изменения. В случае, если вы считаете, что ваши изменения пора внести в основную ветку(main), свяжитесь с другими разработчиками и проконсультируйтесь перед тем, как это делать.
